# artifact-types.md
## Artifacts produced by lesson-planner

Each artifact must carry a measurable objective, a cited+verified standard, differentiation, an
aligned check for understanding, and the metadata block. Templates live in `assets/templates/`.

### Lesson plan
- **Purpose:** a single class period of instruction.
- **Required elements:** objective(s); standard(s); materials; gradual-release sequence (opening /
  I do / we do / you do / closing); check(s) for understanding; differentiation (UDL + tiering + EL
  + IEP); timing; assessment/exit check.
- **Template:** `assets/templates/lesson-plan-template.md`.

### Unit plan
- **Purpose:** a multi-day arc toward a set of standards.
- **Required elements:** unit objectives + standards; sequence of lesson topics with a day-by-day
  pacing table; summative assessment plan; differentiation strategy across the unit.
- **Template:** `assets/templates/unit-plan-template.md`.

### Guided notes
- **Purpose:** scaffolded note-taking that supports access and engagement.
- **Required elements:** the key content with strategic blanks/prompts; aligned to the lesson
  objective; a completed teacher key.

### Exit ticket
- **Purpose:** a brief formative check at lesson close.
- **Required elements:** 1-3 items measuring the objective; an answer key; a "what to do with the
  data" note (who needs reteach / tier-2).
- **Template:** `assets/templates/exit-ticket-template.md`.

### Centers / stations
- **Purpose:** rotating small-group practice.
- **Required elements:** 3-4 stations with directions, materials, and the skill each targets;
  tiering across stations; a management/rotation note.

### Project / performance task
- **Purpose:** applied, transfer-level work.
- **Required elements:** the task + authentic context; standards; success criteria (links to a
  rubric — coordinate with assessment-designer); stages/checkpoints; differentiation.

## Validation (extends `shared/quality/verification-checklists.md`)
- Objective is **measurable** and matches the cited standard's grade + intent.
- The check for understanding actually measures the objective (alignment).
- Differentiation is concrete (named UDL options + at least one tier/EL/IEP support), not generic.
- Reading level matches the band (`shared/quality/readability-age.md`).
