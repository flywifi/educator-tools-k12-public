# artifact-types.md
## Artifacts produced by special-education-support

All outputs are **team drafts** with a legal-boundary + human-review note and placeholder data only.

### Accommodation plan
- **Purpose:** list supports that change *how* a student accesses learning (expectation unchanged).
- **Required elements:** area of need; specific accommodations by category (presentation, response,
  setting, timing, organization); where/when used; the "validate against the actual IEP/504" note.
- **Template:** `assets/templates/accommodation-plan-template.md`.

### Modification plan
- **Purpose:** describe changes to *what* a student is expected to learn (significant team decision).
- **Required elements:** the modified expectations vs. grade-level; linked standards; rationale;
  explicit flag that modifications are a team/legal decision, not assumed.

### IEP goal draft
- **Purpose:** draft SMART goal + present-levels language for the team to refine.
- **Required elements:** baseline (placeholder), measurable goal (condition + behavior + criterion +
  timeframe), how progress is measured; tie to a standard where academic.

### Progress-monitoring tool
- **Purpose:** track movement toward a goal.
- **Required elements:** the skill/goal; a data-collection method + cadence; a simple recording table
  (placeholder student); a decision rule (when to adjust).

## Validation (extends `shared/quality/verification-checklists.md`)
- Accommodation vs. modification is correctly labeled.
- No real student data; legal-boundary + human-review note present.
- Goals are measurable; academic goals cite a real standard.
- Nothing asserts eligibility or a legal determination.
