# artifact-types.md
## Artifacts produced by school-administration

Implementation-level, summarizable, placeholders/aggregates only; metadata block on each.

### Walkthrough instrument
- **Purpose:** quick, focused classroom look-fors tied to a school initiative.
- **Required elements:** the initiative/focus; 3-6 observable look-fors; a simple scale or evidence
  field; a roll-up/summary view for trends across rooms (aggregate, not individual call-outs).
- **Template:** `assets/templates/walkthrough-tool-template.md`.

### Implementation plan
- **Purpose:** roll out an initiative.
- **Required elements:** goal + rationale; phases with milestones + dates; roles/responsibilities;
  supports/PD needed; risks; the data that defines success.

### Monitoring / data plan
- **Purpose:** track whether an initiative is working.
- **Required elements:** lead + lag indicators; data sources + cadence; review routine (who looks at
  what, when); decision rules for adjusting course.

## Validation (extends `shared/quality/verification-checklists.md`)
- Look-fors/indicators are **observable and tied to the stated initiative**.
- Plans have phases, roles, milestones, and a clear definition of success.
- Aggregate framing; no real student/staff PII; summarizable for decisions.
