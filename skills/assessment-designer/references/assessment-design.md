# assessment-design.md
## Assessment design principles
What makes an assessment *valid and fair* (feeds the Alignment, Educational Quality, and
Accessibility gates). Apply during Generation.

## 1. Validity — measure what you intend
Each item must measure the target objective/standard. Build a quick **blueprint**: item → standard →
cognitive level. If an item needs a skill you didn't intend to test (e.g., heavy reading on a math
item), it threatens validity — simplify or scaffold.

## 2. Cognitive balance (DOK / Bloom)
Mix levels deliberately: recall/skill **and** application/reasoning. Avoid all-recall assessments for
standards that demand reasoning. State the intended level per item in the blueprint.

## 3. Clear, bias-free items
One idea per item; unambiguous stems; plausible distractors (for multiple choice); no cultural,
gender, or context bias; inclusive names/contexts (`SECURITY_AND_SAFETY.md`).

## 4. Accessibility & accommodations
Readable for the band (`shared/quality/readability-age.md`); allow multiple means of expression
(UDL); support accommodations (extended time, read-aloud where reading isn't the construct);
**assess content, not English proficiency** for ELs.

## 5. Always provide scoring
Answer key for selected-response; a rubric or scoring guide for constructed-response/performance.
Make scoring reproducible — two scorers should reach the same result.

## 6. Close the loop
Include a brief "using the data" note: what each result band implies for reteaching or tiering
(ties assessment back to instruction).
