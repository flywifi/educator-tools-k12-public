# english-learners.md
## Differentiation Engine — English Learner (EL) Supports
Canonical source. Supports for multilingual learners so language demand never blocks access to
grade-level content. Language proficiency is distinct from content ability.

---

## 1. Proficiency levels (WIDA-style, generalized)
Entering → Emerging → Developing → Expanding → Bridging → Reaching. Supports lighten as proficiency
grows; **content stays grade-level** at every level.

## 2. Core supports
- **Comprehensible input:** visuals, gestures, realia, demonstrations; slower/clear speech; chunked
  text.
- **Vocabulary:** pre-teach key + academic vocabulary; cognates; word banks; labeled visuals.
- **Language production scaffolds:** sentence stems/frames, structured talk (partner → group),
  graphic organizers, modeled responses.
- **Background knowledge:** activate/build context not assumed by the text.
- **L1 as a resource:** allow first-language thinking, glossaries, peer support where possible.
- **Assess content, not just English:** allow alternate demonstration of understanding; separate
  language errors from content errors when scoring.

## 3. The four language domains
Plan supports across **Listening, Speaking, Reading, Writing** — many tasks lean on one or two; name
which the artifact targets and scaffold the rest.

## 4. Rules
- EL supports complement UDL (`udl.md`); they are not the same as IEP accommodations
  (`accommodations-catalog.md`) — a student may need both, one, or neither.
- Keep cognitive demand high; lower the **language** barrier, not the **thinking**.
- Record EL supports in metadata (`differentiation`). Checked by **Accessibility** (QG §31) +
  **Educational Quality** (QG §27).
